## ⚠️ Before ANY page goes live (mandatory QA — from the ConversionLab "never" list)

1. **Fill every `[X]` and 🔒 placeholder with REAL numbers.** The existing site's biggest credibility leak is `0+` counters and reused "200+ developers" across unrelated pages. Synthetic stats and case studies are detected by this buyer ($100K–500K, technical). If a real case study doesn't exist yet, that's a Phase 5 (Trust Infrastructure) blocker — don't fake it.
2. **Set message match per ad.** Each page's H1 must repeat the ad's exact key phrase. Use Dynamic Text Replacement on the search verb (build/launch/create) with a sensible fallback. (Ad-matched headlines have driven +152% CVR in ConversionLab's tests.)
3. **One CTA only.** No "Get Free Consultation" + "Schedule Call" + "Request Quote" stacking (the current site's mistake). Each page above already specifies exactly one primary CTA + one low-commitment text link.
4. **Trust beside the form**, at the moment of decision — logos + one real testimonial.
5. **Shortest viable form.** 3 fields for most; 4 only for enterprise pages (09/10/11) where a qualifier raises lead quality. Email labeled "Work email." No reset button. No phone field. Reduce mobile typing with dropdowns.
6. **Mobile:** the three questions — *Where am I? / What can I do? / Why?* — must be answered in the first mobile viewport on every page.

## Ship each page with its hypothesis and test plan
Each page file states its conversion hypothesis. First tests on a paid funnel should almost always be **message match, CTA clarity, or form friction** — test one isolated change at a time, run to 95% significance, make each winner the next control. Diagnose source-first: if Google converts well and another channel doesn't, the gap is usually the channel's timing, not the page.
